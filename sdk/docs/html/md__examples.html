Example Projects

The examples show how to implement the basic operations of the Lumasonic Decoder.

Dependencies

Besides the LumasonicDecoder static library, the example projects also require:

  • All Platforms: cmake
  • Windows: MSVC for C++
  • Linux: make, Clang or g++
  • macOS: Xcode

Building the Examples

From a terminal, set the current directory to the examples directory of the SDK package.

cd ~/LumasonicDecoderSDK/examples

Decide which example you want to build, and then set the current directory to the example project's subdirectory.

cd decoder

Make a new directory in the example project named build and set it as the current directory.

mkdir build && cd build

In the build directory, create the project files with cmake.

cmake ..

The project files for your platform should now exist in the build directory within the example project directory.

Once built, executables will exist in the build/app directory.

Windows

In the build directory open the solution file in Visual Studio and build in Debug or Release mode.

macOS

In the build directory open the project file in Xcode and build in Debug or Release scheme.

Linux

In the build directory run make to build the project.

Examples Summary

examples/decoder

The decoder example is a simple console application that simulates reading decoded Lumasonic color data with only an instance of LumasonicStereoDecoder.

The reading thread in this example is created manually, and demonstrates how to use your own consumer thread to read and process decoded Lumasonic data.

Note
This example does not utilize an actual realtime audio thread, but simulates the audio thread using the application's main execution thread.

examples/reader

The reader example is a simple console application that uses an instance of LumasonicStereoReader and LumasonicStereoDecoder together to read and process Lumasonic data.

Data is read by a custom listener class that implements the LumasonicStereoColorListener interface.

The reading thread in this example belongs to the reader instance. The reader manages its own internal thread automatically.

Setting the reader's thread mode changes the reader's timing, jitter, and CPU usage characterestics.

examples/player

The player example implements a command line Lumasonic file player.

Use the LumasonicFilePlayer tool to generate an AudioDeviceSettings.xml file for use with the player.

See the detailed description of the LumasonicFilePlayer class for an explanation on how to use the player.
